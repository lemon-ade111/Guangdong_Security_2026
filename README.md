# 🛡️ 广东信息安全与合规学习项目 (2026-2029)

本项目记录了一名武汉科技大学 CS 专业学生，在 AI 时代背景下探索信息安全合规、数据审计及自动化工程的学习历程。

## ⚖️ 学习守则 (Ethical Notes)
- **合规第一**：所有实验均在合法授权或公开数据集（OSINT）下进行，严格遵守《数据安全法》及《个保法》。
- **技术向善**：学习爬虫与审计技术仅用于提升安全防御及合规检测能力，尊重目标平台的 `robots.txt` 协议。

## 📅 2026-03-01 学习日志：从零开始的环境与逻辑构建

今天是计划启动的第一天。虽然遇到不少挫折，但完成了从环境搭建到初步实战的闭环。

### 1. 环境构建与工具调优
- 搭建了 Windows (Trae) + Ubuntu (SSH 远程) 的开发环境。
- 尝试通过 AI (DeepSeek/Gemini) 辅助 Python 脚本编写。感悟：AI 可以大幅提升编码速度，但环境配置与依赖管理（如 `venv`）仍需扎实的 Linux 基础。

### 2. 安全情报自动化尝试 (OSINT Practice)
- **实践过程**：目标是构建一个自动化的安全资讯搜集器。
- **遇到的困难**：先后尝试抓取 CNCERT、Freebuf、安全客等站点。
  - *反思*：深刻体会到了动态渲染（JS Rendering）和 WAF（防火墙）拦截的威力。对于新手来说，`requests` 库在面对复杂防御时显得力不从心。
- **最终方案**：通过对 Sec-Wiki 的结构分析，成功实现了基础数据的抓取与 CSV 存证。

### 3. 行业标准研读 (NIST CSF 2.0)
- 阅读了 NIST 2.0 框架。重点梳理了 **Govern (治理)** 和 **Identify (识别)** 两个模块的专业英语词汇。
- *心得*：合规不仅仅是代码，更是一套管理逻辑。
