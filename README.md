本项目记录了一名武汉科技大学 CS 专业学生，在 AI 时代背景下探索信息安全合规、数据审计及自动化工程的学习历程。

## 📅 学习日志 (Learning Logs)

### 2026-03-03：数据库主权、哈希碰撞与网络排障
**今日重点：从“零散脚本”向“持久化系统”跨越，理解数据完整性的物理实现。**

- **算法底层探索**：
    - **哈希冲突实验**：手动构造 CollidingKey 类，强制引发哈希冲突，观测到 Python 字典查询性能在极端情况下从 O(1) 退化为 O(n) 的过程。
    - **安全启示**：理解了“算法复杂度攻击 (Hash-Flooding DoS)”的原理，意识到防御必须深入到数据结构层面。
- **架构重构：SQLite 存储系统**：
    - **持久化升级**：放弃 CSV 文件，建立了基于 **SQLite** 的工业级审计数据库，实现了 ACID 原则下的数据一致性。
    - **完整性签名**：集成了 Python `hashlib` 模块。在入库前对（标题+链接）进行 **SHA-256 签名**。
    - **时间标准化**：采用 UTC 时间戳存储，建立符合国际合规溯源标准的审计留痕意识。
- **网络工程实战**：
    - **排障记录**：遇到虚拟机 DNS 解析失败与网关不可达故障。
    - **解决方案**：通过重启宿主机 VMware NAT/DHCP 服务，并重置 Ubuntu 网络接口成功恢复通信。建立了“物理链路-网络服务-应用解析”的系统化排障思维。
- **今日感悟**：数据库里的每一行数据，配合其对应的哈希指纹，才是合规审计中不可篡改的“铁证”。

### 2026-03-02：底层逻辑探索与审计逻辑迭代
**今日重点：穿透“黑盒”，从系统与协议底层理解安全。**

- **技术底座**：通过 `id()` 实验验证内存引用机制；通过 `htop` 和 `kill -9` 掌握进程管理权限。
- **协议逆向**：抛弃 `requests` 库，利用原生 `socket` 手撸 HTTP 请求，通过 **TShark** 解析 TCP 三次握手报文。

### 2026-03-01：计划启动与自动化初探
**今日重点：打通跨平台环境，建立 OSINT 情报流。**
