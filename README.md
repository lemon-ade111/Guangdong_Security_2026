# 🛡️ 广东信息安全与合规学习项目 (2026-2029)

本项目记录了一名武汉科技大学 CS 专业学生，在 AI 时代背景下探索信息安全合规、数据审计及自动化工程的学习历程。

---

## 📅 学习日志 (Learning Logs)

### 2026-03-02：底层逻辑探索与审计逻辑迭代
**今日重点：穿透“黑盒”，从系统与协议底层理解安全。**

- **技术底座（系统与网络）**：
    - **内存审计**：通过 `id()` 实验验证了 Python 的小整数池机制，初步理解变量在物理内存中的引用逻辑。
    - **进程管理**：编写 `rogue_process.py` 模拟失控程序，练习使用 `htop` 和 `ps -ef` 定位 PID，并利用 `kill -9` 执行内核级清理。
    - **协议逆向**：抛弃 `requests` 库，利用 Python 原生 `socket` 库手动构造 HTTP 请求。通过 **TShark** 捕获并成功解析 **TCP 三次握手 (SYN, SYN-ACK, ACK)**，直观理解了“传输中数据（Data-in-transit）”的本质。
- **合规审计实战**：
    - **发现问题**：初版审计脚本仅能进行简单的关键字匹配，在面对“URL 字段恶意删减”或“CSV 结构干扰”时完全失效。
    - **逻辑重构**：引入 **Regex（正则表达式）** 校验与 URL 连通性测试。对标 **PIPL（个保法）第 51 条**，实现了对数据完整性（Integrity）的初步自动化校验。
- **今日感悟**：AI 编写的代码需要人工进行逻辑“兜底”。理解了底层协议，审计才会有真正的“深度”。

### 2026-03-01：计划启动与自动化初探
**今日重点：打通跨平台环境，建立 OSINT 情报流。**

- **环境构建**：完成了 Windows (Trae) + Ubuntu (SSH 远程) + Git 的复合开发环境搭建。
- **情报自动化 (OSINT)**：
    - 目标：构建自动化的安全通报搜集器。
    - 挑战：先后尝试抓取 CNCERT、Freebuf、安全客等站点，深刻体会到了动态渲染（JS Rendering）和 WAF 防护的拦截机制。
    - 最终实现：通过分析 Sec-Wiki 的静态结构，成功打通了数据抓取与 CSV 存证的闭环。
- **行业标准研读**：初步拆解 **NIST CSF 2.0** 框架，建立了关于 **Govern (治理)** 和 **Identify (识别)** 的专业术语映射表。
- **今日感悟**：迈出从 0 到 1 的第一步，意识到环境配置与依赖管理（venv）是工业级开发的基础。
